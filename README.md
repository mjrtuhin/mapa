# MAPA - Map Analytics

A marketing intelligence platform that analyzes Google Maps data to identify business opportunities, market gaps, and predict success for new businesses in any city.

Type a business category (restaurants, gyms, salons, grocery stores, anything) and a city name. MAPA crawls Google Maps to collect every listing, scrapes customer reviews, analyzes them with AI, and tells you where the market is underserved, where it is oversaturated, and whether a new business at a given location would likely succeed.

## What It Does

**Phase 1 -- Dashboard and Crawlers**

- Searches an entire city for all businesses of a given type using grid-based area splitting
- Collects customer reviews beyond the 5-review API limit using browser automation
- Analyzes reviews with AI (Groq) to extract sentiment, common topics, praised aspects, and complaints
- Detects market gaps (unmet customer needs) and saturation points across the area
- Maps nearby context for each business: parking, public transport, schools, main roads, shopping areas, hospitals, parks, tourist attractions
- Displays everything in an interactive Streamlit dashboard with maps, charts, and comparison tables

**Phase 2 -- Machine Learning**

- Trains a model to predict whether a new business would achieve 4.1+ stars based on its location features
- Compares Logistic Regression, Random Forest, and XGBoost
- Features include binary indicators (has_parking, near_school, on_main_road) and numeric values (amenity counts, distances)
- Saves the best model for integration back into the dashboard

## Tech Stack

- **Dashboard:** Streamlit
- **Crawlers:** SeleniumBase (free, primary) + Google Places API (optional)
- **AI Analysis:** Groq API with Llama 3.3 70B
- **ML Models:** scikit-learn, XGBoost
- **Maps:** Folium
- **Charts:** Plotly
- **Geocoding:** Geopy (Nominatim)

## Setup

**1. Clone the repository**

```
git clone https://github.com/mjrtuhin/mapa.git
cd mapa
```

**2. Install dependencies**

```
pip install -r requirements.txt
```

**3. Set up API keys**

Copy the example env file and add your keys:

```
cp .env.example .env
```

Open `.env` and add:

- `GROQ_API_KEY` -- Required for AI review analysis. Get a free key at https://console.groq.com
- `GOOGLE_MAPS_API_KEY` -- Optional. Only needed if you want to use the Google Places API instead of Selenium.

**4. Run the dashboard**

```
streamlit run app.py
```

**5. (Optional) Run the ML notebook**

First, collect some data using the dashboard. Then open the notebook:

```
jupyter notebook notebooks/business_success_predictor.ipynb
```

## How to Use

1. Open the dashboard in your browser (Streamlit will show the URL after running)
2. In the sidebar, type a business type (e.g., "Restaurants") and a city (e.g., "Birmingham")
3. Choose your data source: Selenium (free) or Google Places API
4. Set how many reviews to collect per business
5. Optionally enable "Collect Nearby Context" for amenity data
6. Click "Search & Analyse"
7. Browse the five tabs: Overview, Reviews Analysis, Market Gaps, Nearby Context, Success Predictor

## Data Sources

**Selenium (Default, Free)**

Uses SeleniumBase with undetected Chrome mode to scrape Google Maps directly. No API key needed. Slower but collects unlimited reviews. Handles anti-detection, dynamic scrolling, and the February 2026 "limited view" restriction.

**Google Places API (Optional)**

Uses the official API for business listings. More reliable but limited to 5 reviews per business and has usage costs. Requires a Google Cloud account with the Places API enabled.

## Notes

- Selenium scraping speed depends on your internet connection and the number of businesses in the area
- The grid splitting system divides large cities into smaller search cells to overcome Google's result limits
- Collected data is saved as JSON in the `data/` folder for reuse and ML training
- The ML model needs data from multiple searches to train effectively. The more data you collect, the better the predictions.
